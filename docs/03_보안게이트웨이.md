# BoanClaw 보안 게이트웨이

## 게이트 개요

| 게이트 | 위치 | 적용 시점 | 판정 |
|-------|------|---------|------|
| **Input Gate (G1→G2→G3)** | boan-proxy | 모든 S1 방향 정보 흐름 | allow / block / hitl_required / credential_required |
| **Credential Gate (local)** | boan-credential-filter (thin forwarder) | 저장/조회 요청 | cloud gate 로 proxy |
| **Credential Gate (cloud)** | boan-org-credential-gate (Cloud Run + Secret Manager) | LLM 호출 직전 {{CREDENTIAL:}} → 평문 치환 | inject / scrub echo / revoke |
| **LLM Egress Gate** | boan-org-llm-proxy (Cloud Run) | 모든 외부 LLM 호출 | allowlist / deny / rate limit / device JWT |
| **Network Gate (local)** | boan-proxy MITM | 모든 아웃바운드 HTTP | allow (whitelist) / block |

## Input Gate — G1/G2/G3 가드레일

### 적용 범위

| 경로 | G1 | G2/G3 |
|------|-----|-------|
| 키보드 텍스트 (mode: text) | O | ask/deny만 |
| 특수키 (mode: key) | O (safe키 허용) | - |
| 단축키 (mode: chord) | O (safe조합 허용) | - |
| 클립보드 붙여넣기 (mode: paste) | O | ask/deny만 |
| OpenClaw 채팅 (chat/forward) | O | ask/deny만 |
| Computer-use type/key | O | ask/deny만 |
| 파일 S2→S1 전송 | O | ask/deny만 |
| 파일 S1→S2 전송 | X (높→낮 면제) | X |

### 평가 순서

```
입력 수신
    ↓
━━━ G1: 정규식 가드레일 (모든 사용자) ━━━
  credential 패턴 (private key, API token, JWT, password=)
    → 감지 시: credential_required (즉시 차단)
    → 미감지: 계속
    ↓
사용자 정책 확인
    ├─ allow 사용자 → G2/G3 건너뜀 → DLP 검사 → 통과
    └─ ask/deny 사용자 →
        ↓
━━━ G2: 헌법 + 보안 LLM 가드레일 ━━━
  보안 LLM (LLM Registry 의 Security binding — Ollama / Anthropic / OpenAI / 사내 보안 모델)
  이 헌법 기준 판정
    → allow: DLP 검사로
    → block: [G2] 즉시 차단
    → ask: G3으로
        ↓
━━━ G3: 자기진화 Wiki 가드레일 ━━━
  같은 보안 LLM 이 과거 결정 50건 + wiki 노드를 few-shot 으로 보고 재판정
    → allow: DLP 검사로
    → block: [G3] 즉시 차단
    → ask: 인간(소유자) 승인 큐 → Approvals 페이지
        ↓
━━━ DLP 검사 ━━━
  패턴 매칭 + 선택적 LLM 분류
    → allow / redact_required / block
```

### Fail-Closed 동작

| 상황 | 동작 |
|------|------|
| G2 LLM 미설정 | heuristic(정규식) fallback |
| G2 LLM 설정 + 연결 실패 | **block** (fail-closed) |
| G3 LLM 연결 실패 | **block** (fail-closed) |
| guardrail 서버 미연결 | **block** (fail-closed) |
| credential-filter 미응답 | **block** (placeholder resolve 실패) |
| credential-gate (cloud) 미응답 | **block** (org-llm-proxy 가 {{CREDENTIAL:*}} 치환 실패 → upstream 호출 안 함) |
| org-llm-proxy 미응답 / 네트워크 단절 | **block** (외부 LLM 경로 자체가 없음, 로컬 직통 없음) |
| device JWT 무효 (P3) | **401 unauthorized** (Cloud Run 에서 즉시 거부) |
| device 리볼드 (P4) | **403 "device revoked"** |
| OpenClaw 무결성 실패 | **sandbox 컨테이너 즉시 종료** (entrypoint sha256/version 검증) |

## Credential Gate — S5 Cloud 분리

```
구조 (P2 완료):
  boan-org-credential-gate (Cloud Run + GCP Secret Manager)
    - 실제 credential 평문은 Secret Manager 에만 존재
    - secret 이름: boan-cred-{org}-{role}
    - labels: managed-by=boan-org-credential-gate, boan-org={org}, boan-role={role}
    - 접근: bearer 토큰 + (옵션) device JWT
    - 응답: 메타데이터만 또는 /v1/resolve 호출 시 평문 반환 (호출자: org-llm-proxy)

  boan-credential-filter (S3, thin forwarder 모드)
    - BOAN_ORG_CREDENTIAL_GATE_URL 설정 시 register/get/list/revoke 전부 cloud 로 proxy
    - 로컬 디스크에 평문/AES ciphertext 저장 없음
    - legacy local 모드는 credentials.json 파일 있을 때만 fallback (P2 이후 사용 안 됨)

  boan-proxy (S3)
    - {{CREDENTIAL:role}} placeholder 만 다룸
    - resolveTemplateCredentials 가 OrgLLMProxyURL 설정 시 placeholder 그대로 passthrough
    - 실제 치환은 cloud 에서만

프롬프트 처리:
  G1/G2 가드레일 통과 → envelope {org, headers{...}, body_b64} 로 org-llm-proxy 에 전송
  → credresolver.ResolveAll 가 {{CREDENTIAL:role}} 탐지 → gate /v1/resolve 호출 → 평문 substitute
  → upstream LLM 호출
  → credresolver.ScrubEchoes 가 response body 에서 credential 문자열 echo 탐지 → [REDACTED]
  → caller (boan-proxy) 에 반환

API (local):
  POST   http://boan-credential-filter:8082/credential/{org}   (register, forwards to cloud)
  GET    http://boan-credential-filter:8082/credential/{org}    (list metadata, forwards)
  GET    http://boan-credential-filter:8082/credential/{org}/{role}   (get plaintext, calls /v1/resolve)
  DELETE http://boan-credential-filter:8082/credential/{org}/{role}

API (cloud — org-llm-proxy/credential-filter only):
  POST   https://boan-org-credential-gate-{org}-*.a.run.app/v1/credentials/{org}
  GET    .../v1/credentials/{org}
  POST   .../v1/credentials/{org}/{role}/revoke    (P4: 감사 이벤트 + 삭제)
  POST   .../v1/resolve                            (평문 반환, audit 이벤트 emit)
```

## LLM Egress Gate — S5 Cloud

```
구조 (P1 완료):
  boan-org-llm-proxy (Cloud Run, org 별 1개)
    - 모든 외부 LLM 호출의 단일 egress
    - upstream host allowlist: ollama.com, api.anthropic.com, api.openai.com, generativelanguage.googleapis.com
    - bearer 토큰 + (P3) Ed25519 device JWT 이중 인증
    - (P4) per-device rate limit (default 120 rpm)
    - (P4) 구조화 JSON 감사 로그 → Cloud Logging

API:
  POST   https://boan-org-llm-proxy-{org}-*.a.run.app/v1/forward
         Headers: Authorization: Bearer {token}, X-Boan-Device-JWT: {EdDSA JWT}
         Body: {org_id, caller_id, target, method, headers, body_b64, timeout_ms}
```

> P1-P4 전체 설계는 [`05_Credential_Vault.md`](05_Credential_Vault.md) 참고.

## Network Gate

```
HTTP 요청 → 정책 allowlist (host:port:method)
  매칭 → 통과
  미매칭 → 403 차단 (fail-closed)
  CONNECT 터널 → 전부 차단
```
