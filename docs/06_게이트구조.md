## 2. 게이트별 역할

### Input Gate (S3→S2 경계, UI 레이어)

- **위치**: boan-proxy `/api/input-gate/evaluate`
- **대상**: 사용자 키보드 입력, 클립보드 붙여넣기 (MyGCP Guacamole 전송 전)
- **동작**:
  - text / key / paste 모드
  - DLP 룰 검사
  - Guardrail 판정
  - 허용된 것만 Guacamole → RDP → GCP Windows VM으로 전달
- **구현 파일**: `internal/proxy/input_gate.go`

### Credential Gate (현재 구현 상태)

- **위치 A**: boan-onecli (S2 sandbox 내부)
  - LLM API 호출 직전 실제 credential 헤더 주입
  - credential-filter에서 TTL 캐싱으로 조회
  - **구현 파일**: `boan-onecli/internal/credclient/client.go`, `gateway.go`

- **위치 B**: boan-proxy DLP (S2→S1 경계)
  - 프롬프트 통과 시 regex로 secret-like 값 `[REDACTED]` 처리
  - **구현 파일**: `internal/proxy/openclaw_provider.go::sanitizeCredentialReadableText`

- **아직 없는 것**:
  - 등록된 credential 값이면 `{{CREDENTIAL:name}}`으로 치환
  - 처음 보는 credential 감지 시 HITL (등록 여부 묻기)
  - 거절 fingerprint 영속 저장

### Network Gate (S2→S1 경계)

- **위치**: boan-proxy HTTP_PROXY
- **동작**: allowlist에 없는 목적지 차단 (fail-closed)
- **구현 파일**: `internal/network/gate.go`, `internal/proxy/proxy.go`

### AI Guardrail Gate (S2→S1 경계)

- **위치**: boan-proxy, boan-policy-server
- **동작**:
  - DLP 검사 (rule-based + local LLM)
  - policy-server에 `allow / ask / block` 판정 요청
  - ask면 HITL approval 생성
- **구현 파일**: `internal/guardrail/client.go`, `internal/dlp/`

## 4. onecli vs boan-proxy 역할 구분

| | onecli | boan-proxy |
|---|---|---|
| 경로 | sandbox → external LLM (직접 호출) | sandbox → HTTP_PROXY → 외부 전반 |
| credential | 실행 직전 실제 값 헤더 주입 | 프롬프트 본문에서 secret 제거 |
| 검사 대상 | HTTP 헤더만 | 프롬프트 본문 + URL + 목적지 |
| 역할 | Credential 주입기 | S2→S1 게이트 |

## 6. Credential Gate 문제점과 목표 흐름

### 현재 문제점

1. `unknown secret`이 generic redaction이 아니라 placeholder처럼 보이는 문자열로 바뀌는 경로가 있었다.
2. OpenClaw 응답/툴 출력에도 같은 값이 다시 사람 readable 형태로 나타날 수 있었다.
3. `{{CREDENTIAL:name}}` 같은 명시적 참조와 generic secret redaction이 한 경로에서 섞여 혼동을 만들었다.
4. 실제 secret 주입 위치가 `onecli` 패턴처럼 한 곳으로 인식되지 않고, prompt 단계와 실행 단계가 섞여 보였다.

### 목표 흐름

```text
사람 입력 / tool output
  -> Credential Gate
     - 등록된 credential 실제 값이면 {{CREDENTIAL:name}} 로 치환
     - 미등록 raw secret이면 [REDACTED] + HITL
  -> secret-free zone
     - LLM prompt
     - sandbox 명령 계획
     - tool 결과 요약
  -> 실행 직전 주입
     - onecli / header injection 경로에서만 실제 secret 사용
  -> 결과 반환 전 재마스킹
     - 사람 readable output
     - LLM readable output
```

### 현재 원칙

- `known registered secret` -> `{{CREDENTIAL:name}}`
- `unknown secret` -> `[REDACTED]`
- 실제 secret은 header 주입 같은 실행 직전 경로에만 존재해야 한다
- 사람/LLM이 보는 모든 surface는 secret-free여야 한다
