# 에이전트 보안 고려사항

Anthropic "Building effective agents" (managed-agents) 기반 보안 분석 + BoanClaw 대응 현황.

---

## 1. Anthropic이 지적한 핵심 위험

### 1-1. Prompt Injection → Credential 탈취

> "untrusted code가 credential과 같은 컨테이너에서 실행되면, prompt injection 한 번으로 환경변수를 읽어 토큰을 탈취할 수 있다"

공격자가 토큰을 획득하면 **새로운 무제한 세션을 생성**하여 에이전트를 완전히 장악.

**BoanClaw 대응:**
- `{{CREDENTIAL:name}}` 플레이스홀더 체계 — 실제 키는 S4 credential-filter (독립 컨테이너) 에만 존재
- sandbox(S2)에서 실행되는 코드는 실제 키에 접근 불가 — `boan-cred-*` 볼륨이 sandbox 에 마운트되지 않음
- 환경변수 삭제 (SSLKEYLOGFILE, NODE_OPTIONS 등 12종)
- `.env` 파일 `/dev/null` 마운트
- **credential gate**: 미등록 키 감지 시 REDACT + HITL 승인
- **OpenClaw supply-chain 검증**: 빌드 시점에 핀된 버전 + sha256, 런타임에 재검사 (fail-closed)

### 1-2. 구조적 분리 (Structural Separation)

> "permission scoping에 의존하지 말고, 구조적으로 토큰이 sandbox에서 접근 불가능하게 만들어라"

Anthropic의 해법: **코드 실행 환경과 credential을 물리적으로 분리.**

**BoanClaw 대응:**
- S4/S3/S2/S1 4단계 영역 분리 — credential은 **S4 (독립 컨테이너)**, 실행은 S2, 외부는 S1
- `boan-credential-filter` 가 별도 컨테이너로 AES 암호화 저장 (`/etc/boan-cred`, `/data/credentials`)
- sandbox 의 docker-compose volumes 에 `boan-cred-*` 절대 추가 안 함 (코드 레벨 강제)
- 실행 시점에만 메모리에 주입, 디스크에 안 씀
- LLM 프롬프트/응답/로그에 실제 키 노출 불가 (placeholder 만 남김)

## 2. Anthropic 권장 패턴과 BoanClaw 매핑

### 2-1. Brain / Hands / Session 분리

Anthropic의 3요소 분리:

| 요소 | 역할 | 보안 경계 |
|------|------|---------|
| **Brain** | Claude + harness — 추론 | credential 접근 불가 |
| **Hands** | sandbox/tools — 코드 실행 | untrusted 환경 |
| **Session** | event log — 상태 저장 | 실행 환경 외부 |

**BoanClaw 매핑:**

| Anthropic | BoanClaw | 위치 |
|-----------|----------|------|
| Brain | OpenClaw + boan-agent | S2 (sandbox) |
| Hands | GCP workstation + computer-use | S1 (external) |
| Session | audit-agent + policy-server | S4 (격리) |
| Credential Vault | boan-credential-filter | **S4 (독립 컨테이너, sandbox 마운트 금지)** |

**핵심 차이**: BoanClaw는 Brain(S2)과 Hands(S1) 사이에 **3-Gate 시스템**(Input/Credential/Network)을 배치. Anthropic은 Brain↔Hands 간 credential 격리만 언급하지만, BoanClaw는 **정보 흐름 자체를 통제**.

### 2-2. Git/Repository 접근 패턴

Anthropic: 토큰으로 clone 후 토큰 폐기. git push/pull은 로컬 설정으로.

**BoanClaw 대응:**
- git-guard.sh: 위험 명령 차단 (reset --hard, push --force, rebase -i, commit --amend)
- SSRF 방어: `safeFetch()`로 사설 IP, 메타데이터 서버 차단
- git 바이너리 래핑: `/usr/bin/git` → `/usr/local/bin/git-guard` → `/usr/bin/git-real`

### 2-3. OAuth/API 프록시 패턴

Anthropic: OAuth 토큰을 vault에 저장, 전용 proxy가 인증 처리.

**BoanClaw 대응:**
- boan-credential-filter: AES 암호화 vault
- boan-proxy: MITM 프록시가 credential 주입
- LLM Registry: 모델 등록 시 키 자동 감지 → credential-filter에 저장
- 모든 LLM 호출: GCP LLM Proxy 경유 (`X-LLM-Target` 헤더로 라우팅)

## 3. Anthropic이 언급하지 않은 영역 — BoanClaw의 추가 보안

Anthropic 문서는 **credential 격리**에 집중. 아래 영역은 언급 없지만 BoanClaw가 자체 구현:

### 3-1. 정보 흐름 통제 (Information Flow Control)

Anthropic은 "무엇을 실행하는가"를 통제하지 않음. BoanClaw는 **"어디로 정보가 흐르는가"를 통제**:

```
높은 영역 → 낮은 영역: 3-Tier 가드레일 검사
낮은 영역 → 높은 영역: 무조건 통과
```

에이전트가 credential 없이도 **데이터를 외부로 유출**하는 시나리오를 방어:
- 키보드/클립보드/채팅/파일 전송 모두 Input Gate 검사
- computer-use type/key 액션도 가드레일 적용

### 3-2. 사용자 신뢰 모델 (User Trust Model)

Anthropic은 에이전트 단위 permission만 논의. BoanClaw는 **사용자 단위 신뢰 레벨**:

- **Allow**: 낮은 흐름 허용, 모니터링만
- **Ask**: 가드레일 적용, 사전차단 + 모니터링
- **Deny**: 낮은 흐름 완전 차단

같은 가드레일 판정이라도 사용자 정책에 따라 결과가 다름.

### 3-3. 자기진화 가드레일 (Self-Evolving Guardrail)

Anthropic은 정적 규칙만 언급. BoanClaw는 **LLM 기반 3-Tier 가드레일**:

1. **Tier 1 헌법**: 보안 LLM이 헌법 기준으로 판정
2. **Tier 2 Wiki**: 과거 결정을 학습한 자기진화 LLM
3. **Tier 3 인간**: 소유자 승인 → wiki 학습 → 헌법 개정 제안

### 3-4. 네트워크 수준 격리 (Network-Level Isolation)

Anthropic은 sandbox 격리만 언급. BoanClaw는 **네트워크 게이트웨이**:

- fail-closed 화이트리스트 (등록 안 된 호스트 전부 차단)
- MITM 프록시로 모든 HTTPS 트래픽 검사
- CONNECT 터널 전면 차단

### 3-5. 감사 추적 (Audit Trail)

Anthropic은 session event log만 언급. BoanClaw는 **전 이벤트 감사**:

- 모든 Input Gate 판정 기록
- 네트워크 차단 이벤트 기록
- credential 사용 이벤트 기록
- HITL 결정 이력 (training log)
- CSP(S4)로 전송하여 변조 방지

### 3-6. Human-in-the-Loop (HITL)

Anthropic: 미언급. BoanClaw:

- 가드레일 ask 판정 → 소유자 Approval 큐
- credential 미등록 키 감지 → HITL 승인
- 헌법 개정안 → 소유자 diff 검토
- 자동 모드: LLM 에이전트가 즉시 판단 (결정 이력 누적)

---

## 5. 결론

Anthropic의 managed-agents 문서는 **credential 격리**라는 핵심 원칙을 제시하며 이는 모든 에이전트 시스템의 기본이다. BoanClaw는 이 원칙을 완전히 수용하면서, Anthropic이 다루지 않은 **정보 흐름 통제**, **사용자 신뢰 모델**, **자기진화 가드레일**, **네트워크 격리**, **HITL 워크플로**를 추가로 구현한다.

핵심 차이: Anthropic은 "에이전트가 credential에 접근하지 못하게" 하는 데 집중. BoanClaw는 거기에 더해 "에이전트가 **무엇을 어디로** 보내는지"까지 통제한다.
