## 4. Credentials

| ID | 테스트 | 방법 | 기대 결과 |
|----|-------|------|---------|
| CR-1 | Organization 키 추가 | Role + Key 입력 → 저장 | 목록 표시, 상태 ok. 저장은 **GCP Secret Manager** (`boan-cred-{org}-{role}`), 로컬 디스크 변동 없음 |
| CR-2 | Organization 키 삭제 | revoke 클릭 | 목록 제거, 이후 `/v1/resolve` 응답 404, `credential_revoke` 감사 이벤트 emit |
| CR-3 | 키 마스킹 표시 | 키 입력 시 미리보기 | `sk-ant-...xxxx` 형태 |
| CR-4 | Personal 탭 조회 | Personal 탭 클릭 | 개인 credential 목록. Organization recommendation critical key 미등록이면 표시 |
| CR-5 | Passthrough 등록 | 이름+값 입력 → 등록 | REDACT 없이 통과 |
| CR-6 | 평문 로컬 저장 검증 | 새 credential 등록 후 `docker exec boan-credential-filter ls /data/credentials/` | 파일 변동 없음 (gate 모드) |
| CR-7 | credential 해석은 cloud 에서만 | LLM 호출 시 boan-proxy 로그 확인 | `resolveTemplateCredentials` 가 placeholder 그대로 passthrough (치환 안 함) |
| CR-8 | Cloud Run 감사 이벤트 | Credentials 페이지에서 register/resolve/revoke | Cloud Logging 에 `credential_put` / `resolve_ok` / `credential_revoke` JSON 이벤트 |

## 15. 자동화 테스트

| 스크립트 | 범위 |
|---------|------|
| `scripts/test.sh` | 전체 (Go unit + TS unit + 통합) — 단일 진입점 |
| `scripts/rebuild.sh` | proxy/sandbox/console 동시 빌드 (메모리 규칙) |

> 이전에 있던 개별 테스트 스크립트(`test-access-level-api.sh`, `test-network-policy.sh`, `test-file-transfer.sh`, `test-computer-use.sh`)는 제거됨 — 모두 `scripts/test.sh` 에 통합.

```bash

go test ./internal/proxy/ -run "G1\|G2\|G3\|InputGate" -v

# Access Level (9개)
go test ./internal/userstore/ -v

# 전체
go test ./... -count=1
```
