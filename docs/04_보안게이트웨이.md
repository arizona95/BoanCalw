# BoanClaw 보안 게이트웨이

## 게이트 개요

| 게이트 | 위치 | 적용 시점 | 판정 |
|-------|------|---------|------|
| **Input Gate (G1→G2→G3)** | boan-proxy | 모든 S1 방향 정보 흐름 | allow / block / hitl_required / credential_required |
| **Credential Gate** | boan-credential-filter (S4 독립) | LLM 호출 시 프롬프트 | 치환 / REDACT / HITL |
| **Network Gate** | boan-proxy MITM | 모든 아웃바운드 HTTP | allow (whitelist) / block |

## Input Gate — G1/G2/G3 가드레일

### 적용 범위

| 경로 | G1 | G2/G3 |
|------|-----|-------|
| 키보드 텍스트 (mode: text) | O | ask/deny만 |
| 특수키 (mode: key) | O (safe키 허용) | - |
| 단축키 (mode: chord) | O (safe조합 허용) | - |
| 클립보드 붙여넣기 (mode: paste) | O | ask/deny만 |
| OpenClaw 채팅 (chat/forward) | O | ask/deny만 |
| Computer-use type/key | O | ask/deny만 |
| 파일 S2→S1 전송 | O | ask/deny만 |
| 파일 S1→S2 전송 | X (높→낮 면제) | X |

### 평가 순서

```
입력 수신
    ↓
━━━ G1: 정규식 가드레일 (모든 사용자) ━━━
  credential 패턴 (private key, API token, JWT, password=)
    → 감지 시: credential_required (즉시 차단)
    → 미감지: 계속
    ↓
사용자 정책 확인
    ├─ allow 사용자 → G2/G3 건너뜀 → DLP 검사 → 통과
    └─ ask/deny 사용자 →
        ↓
━━━ G2: 헌법 + LLM 가드레일 ━━━
  보안 LLM (LLM Registry의 Security binding)이 헌법 기준 판정
    → allow: DLP 검사로
    → block: [G2] 즉시 차단
    → ask: G3으로
        ↓
━━━ G3: Wiki 적응형 가드레일 ━━━
  과거 결정 50건을 학습한 자기진화 LLM
    → allow: DLP 검사로
    → block: [G3] 즉시 차단
    → ask: 인간(소유자) 승인 큐 → Approvals 페이지
        ↓
━━━ DLP 검사 ━━━
  패턴 매칭 + 선택적 LLM 분류
    → allow / redact_required / block
```

### Fail-Closed 동작

| 상황 | 동작 |
|------|------|
| G2 LLM 미설정 | heuristic(정규식) fallback |
| G2 LLM 설정 + 연결 실패 | **block** (fail-closed) |
| G3 LLM 연결 실패 | **block** (fail-closed) |
| guardrail 서버 미연결 | **block** (fail-closed) |
| credential-filter 미응답 | **block** (placeholder resolve 실패) |
| OpenClaw 무결성 실패 | **sandbox 컨테이너 즉시 종료** (entrypoint sha256/version 검증) |

## Credential Gate — S4 분리

```
구조 (Phase 1 완료):
  boan-credential-filter → 독립 컨테이너 (S4)
  AES 키 (/etc/boan-cred/aes.key), 암호화 데이터 (/data/credentials/) → S4 컨테이너에만 마운트
  sandbox(S2) → HTTP API 호출만 가능 (/etc/boan-cred/ 자체가 sandbox 에 없음)

프롬프트 처리:
  등록된 키 치환: sk-ant-... → {{CREDENTIAL:name}}
  미등록 키 감지 → REDACT + HITL 승인
  실행 시점: {{CREDENTIAL:name}} → 실제 키 주입 (메모리만, 디스크 안 씀, 로그에는 placeholder 만)

API:
  GET    http://boan-credential-filter:8082/credential/{org}/{name}
  POST   http://boan-credential-filter:8082/credential/{org}
  DELETE http://boan-credential-filter:8082/credential/{org}/{name}
```

> Phase 2 (GCP Cloud KMS envelope encryption) 은 [`docs/09_Credential_Vault_설계.md`](09_Credential_Vault_설계.md) 참고.

## Network Gate

```
HTTP 요청 → 정책 allowlist (host:port:method)
  매칭 → 통과
  미매칭 → 403 차단 (fail-closed)
  CONNECT 터널 → 전부 차단
```
