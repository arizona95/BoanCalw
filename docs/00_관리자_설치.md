# BoanClaw — 관리자 (조직 소유자) 설치

> 이 문서는 **조직 1개를 GCP 위에 처음 올리는 일회성 작업** 입니다.
> 일반 사용자는 [메인 README](../README.md) 의 한 줄 설치만 따라하면 됩니다.

조직 관리자는 다음을 합니다:
1. GCP 위에 정책서버 / 자격증명 게이트 / LLM egress 브로커를 배포 (Cloud Run + Secret Manager)
2. 사용자에게 한 줄 설치 명령 (`Org URL + Token`) 을 전달
3. 사용자별 access_level (allow/ask/deny) 부여, G2 보안 LLM 헌법 작성, G3 wiki seeding

---

## ⚡ 한 줄 명령

```bash
git clone https://github.com/arizona95/BoanCalw.git ~/boanclaw && cd ~/boanclaw && bash admin-install.sh
```

스크립트가 8 단계를 자동으로 진행합니다:

| 단계 | 내용 |
|---|---|
| 1 | `gcloud` / `terraform` / `openssl` 설치 확인 |
| 2 | `gcloud auth login` (브라우저 OAuth) |
| 3 | GCP 프로젝트 확인 — 없으면 생성 제안 |
| 4 | 결제 계정 연결 확인 |
| 5 | 필수 API 활성화 (Cloud Run / Compute / Secret Manager / IAM / CloudBuild) |
| 6 | IAM 역할 점검 (Project Owner 또는 Editor 권한 필요) |
| 7 | `terraform apply` — `boan-policy-server` / `boan-org-credential-gate` / `boan-org-llm-proxy` 배포 |
| 8 | **Org URL + Token** 출력 → 사용자에게 한 줄 설치 명령과 함께 전달 |

## 인자 / 환경 변수로 한 번에

```bash
./admin-install.sh \
  --project-id sds-boanclaw-prod \
  --org-id sds-corp \
  --owner-email admin@sds.com \
  --region asia-northeast3 \
  --allowed-email-domains samsung.com,samsungsds.com
```

| 인자 | 기본값 | 의미 |
|---|---|---|
| `--project-id` | (필수) | GCP 프로젝트 ID |
| `--org-id` | (필수) | 조직 식별자 (slug, e.g. `sds-corp`) |
| `--owner-email` | (필수) | primary owner 이메일 (관리 콘솔 첫 로그인 계정) |
| `--region` | `asia-northeast3` | Cloud Run 배포 region |
| `--allowed-email-domains` | (필수) | SSO 로그인 허용 도메인 (콤마 구분) |
| `--billing-account` | (선택) | 결제 계정 ID (없으면 대화형 선택) |
| `--yes` / `-y` | off | 모든 확인 프롬프트 자동 수락 |

## 배포되는 GCP 리소스

| 리소스 | 이름 패턴 | 역할 |
|---|---|---|
| Cloud Run service | `boan-policy-server-{org}` | 조직 정책 / 사용자 / 가드레일 / wiki API |
| Cloud Run service | `boan-org-credential-gate-{org}` | credential 평문 vault (Secret Manager 게이트웨이) |
| Cloud Run service | `boan-org-llm-proxy-{org}` | 외부 LLM 단일 egress + credential substitute |
| Secret Manager | `boan-cred-{org}-{role}` | 실제 credential 평문 (조직별/role별 1개) |
| IAM service account | `boan-{org}@{project}.iam.gserviceaccount.com` | Cloud Run 컨테이너 SA (Secret Manager 접근) |

전부 region 1 군데 (`asia-northeast3` 기본) 에 묶여서 배포됩니다.

## 사용자 배포 후 관리자가 할 일 (관리 콘솔)

본인 PC 에서도 `bash install.sh` 로 사용자 컨테이너를 띄워서 관리 콘솔 (`http://localhost:19080`) 로 접속:

1. **Owner 첫 로그인** — TOFU 로 owner IP 자동 캡처 (org-server 가 기록).
2. **사용자 추가** — Users 탭에서 `+` → 이메일 입력 → role 선택. 이메일 도메인이 `--allowed-email-domains` 에 포함돼야 SSO 로그인 가능.
3. **G2 보안 LLM 등록** — LLM Registry 탭에서 모델 추가 (현재 supported: Ollama / Anthropic / OpenAI 호환 endpoint) → "Bind Security" 클릭. 이 모델이 G2 헌법 평가자가 됩니다.
4. **G2 헌법 작성** — Gateway > Guardrail 탭에서 헌법 텍스트 작성. 짧고 구체적일수록 LLM 일관성 ↑.
5. **G3 Wiki seeding** — Wiki Graph 탭에서 `agentic_iterate` 1-2 회 클릭 → 정책 ambiguity 들이 노드로 자동 생성됨. 사용자/관리자 응답이 누적되면 G3 가 G1/G2 개정안 자동 제안.
6. **Credential 등록** — Credentials 탭에서 조직 공용 키 등록 (예: `anthropic-apikey`, `openai-apikey`). credential 평문은 클라이언트 → credential-filter → cloud gate → Secret Manager 까지 한 번 흘러간 후 로컬에 흔적 남지 않음.

## 업데이트 / 재배포

GCP 인프라만 갱신:
```bash
cd ~/boanclaw && git pull && bash admin-install.sh --yes
```
terraform 이 변경된 리소스만 재적용합니다. 기존 Secret 과 사용자 데이터는 보존.

## 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| `gcloud auth login` 후 권한 부족 에러 | 해당 프로젝트의 Owner / Editor / Cloud Run Admin 역할 필요 |
| `terraform apply` 가 quota 초과로 실패 | Cloud Run / Secret Manager 콘솔에서 quota request, 또는 다른 region 시도 |
| Cloud Run service 가 cold start 너무 느림 | `gcloud run services update boan-policy-server-{org} --min-instances=1` 로 warm 유지 (월 약 $5 추가) |
| Secret Manager `permission denied` (사용자 시점) | install.sh 출력의 `BOAN_ORG_TOKEN` 이 누락/오타. 관리 콘솔의 Org Settings 탭에서 token 회전 가능. |

## 보안 모델 요약

- **로컬 호스트가 root 탈취 당해도** credential 평문은 Cloud Run 메모리 (수 ms 만 존재) 외에는 어디에도 없습니다. 공격자는 최대 device JWT (P3) 또는 bearer token 만 획득 가능.
- **device JWT (P3, P4)** — 로컬이 매 호출마다 Ed25519 서명 → Cloud Run 가 검증. bearer 만 탈취된 시나리오에서도 디바이스 키 없으면 호출 불가.
- **revoke** — Cloud Run env `BOAN_REVOKED_DEVICES` 추가만으로 즉시 차단 (재배포 X).
- **rate limit** — 디바이스당 120 rpm (default), runaway agent 가 비용/유출 폭주 못 함.

자세한 내용은 [`05_Credential_Vault.md`](05_Credential_Vault.md) 참고.
