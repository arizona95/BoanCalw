# BoanClaw 보안 요건

## 정책 서버 (S4)

| ID | 요건 | 구현 |
|----|------|------|
| BR-01 | 정책 ed25519 서명 검증, 응답 없으면 fail-closed | policy-server + signing 모듈 |
| BR-02 | 정책 버전 관리 (최대 10버전, 롤백 지원) | policy.Store, v{N}.json |
| BR-08 | BoanClaw 런타임 자기 버전 검증 (allowlist + min version) | policy-server `checkin()` + `VersionPolicy` |
| BR-09 | 감사 로그 CSP 저장소 전송 | boan-audit-agent |

## Credential Vault (S5 Cloud)

| ID | 요건 | 구현 |
|----|------|------|
| BR-03 | Credential 평문을 로컬 호스트에서 완전 분리 | boan-org-credential-gate (Cloud Run + Secret Manager) |
| BR-04 | 로컬 디스크/볼륨에 평문/AES 키 저장 금지 | credential-filter thin forwarder 모드, 로컬 credentials.json 쓰기 차단 |
| BR-06 | credential 접근은 HTTP API 만 | local: POST /api/credential/v1/store → filter → cloud gate /v1/credentials/{org} |
| BR-07 | GCP Secret Manager 로 envelope encryption | secret 이름 `boan-cred-{org}-{role}`, labels managed-by/org/role |
| BR-10 | Placeholder-only LLM 요청 (평문은 cloud 에서만) | boan-proxy `{{CREDENTIAL:role}}` 유지, Cloud Run 에서 LLM 호출 직전 치환 |
| BR-11 | upstream 응답에서 credential echo scrubbing | org-llm-proxy `credresolver.ScrubEchoes` |
| BR-12 | per-credential revoke + 감사 이벤트 | POST /v1/credentials/{org}/{role}/revoke |

## 외부 LLM Egress (S5 Cloud)

| ID | 요건 | 구현 |
|----|------|------|
| BR-20 | 모든 외부 LLM 호출은 단일 egress broker 경유 | boan-org-llm-proxy (Cloud Run, org 별 1개) |
| BR-21 | upstream host allowlist 강제 | BOAN_ORG_LLM_PROXY_ALLOWED_HOSTS |
| BR-22 | 로컬 호스트에서 ollama/anthropic/openai 로의 직접 egress 금지 | openclaw_provider.dispatchLLMRequest → forwardViaOrgProxy |
| BR-23 | 디바이스별 인증 (static bearer 도난 단독 공격 차단) | P3 Ed25519 device JWT (aud=boan-org-cloud, ttl=5min) |
| BR-24 | 디바이스 긴급 차단 (재배포 없음) | BOAN_REVOKED_DEVICES env + 403 "device revoked" |
| BR-25 | per-device 요청율 제한 (runaway agent 방지) | BOAN_ORG_LLM_PROXY_RPM (default 120 rpm) |
| BR-26 | 구조화 감사 로그 (Cloud Logging 통합) | auditlog.Emit JSON events: forward_ok/error, resolve_ok, credential_put, revoke, auth_reject, rate_limit |

## 게이트 시스템 (S3)

| ID | 요건 | 구현 |
|----|------|------|
| BR-05 | 네트워크 정책 화이트리스트 게이트 (fail-closed) | boan-proxy Network Gate |
| ZR-01 | S-Level 분류 체계 (S1~S5) | 전 컴포넌트 적용. S5=Cloud credential vault+egress broker |
| ZR-02 | 3-Gate 모델 (Input/Credential/Network) | input_gate.go, credential_gate.go, network gate |
| ZR-03 | G1/G2/G3 가드레일 체계 | G1:정규식, G2:헌법+LLM, G3:wiki적응형 |
| ZR-04 | 사용자 신뢰 모델 (allow/ask/deny) | userstore.AccessLevel |
| ZR-05 | SSO 도메인 제한 (동적 변경) | org_settings.allowed_domains |
| ZR-06 | G2/G3 LLM fail-closed | LLM 설정+연결실패 → block |
| ZR-07 | G1 전원 적용 (allow 포함) | allow 사용자도 credential 정규식 차단 |
| ZR-08 | G3→G1/G2 자동 개정 | wiki가 정규식/헌법 diff 제안 → 소유자 수락 |

## Sandbox 보안 (S2)

| ID | 요건 | 구현 |
|----|------|------|
| SB-01 | 환경변수 삭제 (SSLKEYLOGFILE, NODE_OPTIONS 등 12종) | entrypoint.sh |
| SB-02 | .env 파일 /dev/null 마운트 | entrypoint.sh |
| SB-03 | 비root 실행 (`boan` 사용자, host UID 와 동기) | Dockerfile ARG `BOAN_UID/BOAN_GID` + gosu |
| SB-04 | Git 위험 명령 차단 (reset --hard, push --force 등) | git-guard.sh |
| SB-05 | SSRF 방어 (사설 IP, 메타데이터 서버 차단) | boan-agent ssrf.ts |
| SB-06 | Credential 파일 미존재 (/etc/boan-cred 없음) | volume 분리 (compose 가 sandbox 에 마운트 안 함) |
| SB-07 | OpenClaw 버전 핀 + sha256 빌드 검증 | Dockerfile `ARG OPENCLAW_VERSION` + `/opt/boanclaw-meta/openclaw.mjs.sha256` |
| SB-08 | OpenClaw 런타임 무결성 검증 (fail-closed) | entrypoint.sh — sha256 / version 재검사, mismatch 시 컨테이너 종료 |
| SB-09 | OpenClaw 동적 allowlist (정책으로 좁히기) | `BOAN_OPENCLAW_ALLOWED_VERSIONS` env (콤마구분) |
| SB-10 | 호스트↔sandbox UID 동기 (호스트 권한 충돌 방지) | install.sh + compose `BOAN_UID/BOAN_GID` build args |
