## Sandbox 보안 (S2)

| ID | 요건 | 구현 |
|----|------|------|
| SB-01 | 환경변수 삭제 (SSLKEYLOGFILE, NODE_OPTIONS 등 12종) | entrypoint.sh |
| SB-02 | .env 파일 /dev/null 마운트 | entrypoint.sh |
| SB-03 | 비root 실행 (`boan` 사용자, host UID 와 동기) | Dockerfile ARG `BOAN_UID/BOAN_GID` + gosu |
| SB-04 | Git 위험 명령 차단 (reset --hard, push --force 등) | git-guard.sh |
| SB-05 | SSRF 방어 (사설 IP, 메타데이터 서버 차단) | boan-agent ssrf.ts |
| SB-06 | Credential 파일 미존재 (/etc/boan-cred 없음) | volume 분리 (compose 가 sandbox 에 마운트 안 함) |
| SB-07 | OpenClaw 버전 핀 + sha256 빌드 검증 | Dockerfile `ARG OPENCLAW_VERSION` + `/opt/boanclaw-meta/openclaw.mjs.sha256` |
| SB-08 | OpenClaw 런타임 무결성 검증 (fail-closed) | entrypoint.sh — sha256 / version 재검사, mismatch 시 컨테이너 종료 |
| SB-09 | OpenClaw 동적 allowlist (정책으로 좁히기) | `BOAN_OPENCLAW_ALLOWED_VERSIONS` env (콤마구분) |
| SB-10 | 호스트↔sandbox UID 동기 (호스트 권한 충돌 방지) | install.sh + compose `BOAN_UID/BOAN_GID` build args |
